# Распределённое приложение для параллельного решения СЛАУ методом ортогонализации

## Описание проекта

Приложение реализует распределённое решение систем линейных алгебраических уравнений (СЛАУ) в локальной вычислительной сети (ЛВС) с использованием протокола TCP. Разработка выполнена в рамках курсового проекта по дисциплине «Распределённые информационные системы».

В качестве основного метода используется метод ортогонализации — модифицированный алгоритм Грама–Шмидта (Modified Gram-Schmidt, MGS), обладающий повышенной численной устойчивостью и хорошей способностью к распараллеливанию. Для сравнения производительности реализован последовательный метод Гаусса.

## Возможности

- решение СЛАУ методом Гаусса (последовательная версия);
- решение СЛАУ методом ортогонализации MGS (последовательная версия);
- распределённое решение MGS по схеме master-worker с параллельным опросом вычислительных узлов;
- обмен данными между узлами по протоколу TCP с префиксной схемой (длина сообщения + данные);
- чтение входных данных из текстовых файлов:
  - матрица системы;
  - вектор правых частей;
  - список вычислительных узлов (host:port);
- генерация тестовых данных с известным решением;
- нагрузочное тестирование и сравнение производительности методов;
- модульные тесты для проверки корректности компонентов;
- графический интерфейс для удобного управления.

## Структура проекта

## Установка зависимостей

```bash
pip install numpy pytest
```

## Запуск
1. Запуск вычислительных узлов (worker)

В отдельных терминалах запустите необходимое количество worker'ов (рекомендуется 2):

```bash
python worker.py --port 5001
python worker.py --port 5002
```

2. Запуск управляющего узла (master)

```bash
python master.py
```

3. Запуск нагрузочного тестирования

```bash
python benchmark.py
```

4. Запуск модульных тестов

```bash
python -m pytest tests/ -v
```

5. Запуск графического интерфейса

```bash
python app.py
```

### Форматы входных файлов
matrix.txt

Первая строка — размерность n, далее n строк по n чисел через пробел:

```
3
4 1 1
1 5 1
1 1 6
```

vector.txt

Первая строка — размерность n, далее n чисел (каждое с новой строки):

```
3
9
14
21
```

nodes.txt
Список узлов в формате host:port (каждый с новой строки):

```
127.0.0.1:5001
127.0.0.1:5002
```
